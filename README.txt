Веб приложение представляет собой игру, в которой пользователь появляется в панораме со случайными координатами. Он должен
определить, где он находится.
После того, как пользователь выберет место на карте, приложение подсчитывает баллы. Чем меньше разница в расстоянии между точкой подобранной программой
и точкой выбранной пользователем, тем больше баллов.
Веб приложение предоставляет пользователю функционал регистрации, авторизирования. Баллы пользователей и количество матчей заносится в базу данных.
Пользователи могут видеть рейтинги, количество матчей других пользователй. Также они могут увидеть свой рейтинг и количество матчей на личной странице.
На личной страице также отображаются последние игры. Если пользователь - администратор, на личной странице появляется кнопка - войти в админку.
В админке аминистратор может редатировать и удалаять пользователей. Чтобы найти конкретного пользователя, в админке есть формы для поиска.

Используемые сторонние библиоткеи. Для отображения панорам, в проиложении используется JavaScript Api Яндекс карт. Для создания серверной части используется Flask фреймворк. Для общения с базой даннных используется орм SqlAlchemy. Для хеширования паролей werkzeug. Для отображения
форм используется flask_wtf, wtforms. ТАкже используется модуль schedule, для выполнения функций в заднное время.

Приложение разрабатывалось из расчета на теоритически возможное реальное использование. Для этого в странице рейтинг сделан переход между страницами.
Все пользователои не могут отображаться на одной странице, т.к. отображение большого количества пользователй может застопорить работу сайта.
Для тестирования в базе данных было создано около 50000 пользователей со случайными логинами, никами, паролями, количеством матчей.
Вполне реальная ситуация, на сайте одновременно находятся 30-50 пользователей, а зарегистрированно аккаунтов 50000 или больше. Это 
могут быть неактивные аккаунты, аккаунты с одним хозяином, и т.д. Также. вытаскивать из базы данных 50 тыс. пользователей для каждого запроса
очень долго. Поэтому пользоваели сохраняются в оперативную память, и уже оттуда программа их получает. Все изменения с пользователями пишутся в базу
данных. Список пользователей внутри программы обновляется каждую минуту.
